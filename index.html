GIT

Everything you need to know about GIT

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.
Version control system (VCS) is a software tool that helps software teams manage changes to source code over time. It allows multiple developers to
collaborate on the same files, tracks every change, and maintains a record of each modification, including authorship, timestamp, and other details.
Git is easy to learn and practice and also extremely useful to other platforms like Github.
GitHubis a developer platform that allows developers to create, store, manage and share their code. It uses Git software, providing the distributed version control of Git plus access control, bug tracking, software feature requests, task management, continuous integration, and wikis for every project.
There are other alternative Version control softwares e.g SubVersion(SVN), Mercurial, Bitbucket, GitLab, Perforce, Bazaar, Fossil, Darcs.
All this version control softwares are good but Git is a fast, distributed version control system with powerful branching and merging capabilities, making it ideal for collaborative development.

1. Installing Git

First check if git is installed in your machine. To check that open your command prompt and type:

git --version

Checking Git version

If Git is not installed, download and install it from Git's official website.

2. Configuring Git

Next step is to configure your Git username and email by typing the following commands in your command prompt: git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

3. Initializing a Git Repository

A Git repository is a directory where Git tracks changes to files. To initalize your repository open your command prompt in the directory where you have stored the files and type: git init

Initializing a Git repository

4. Adding Files to the Staging Area

The staging area is where you prepare files to be committed. Type this in the command prompt:

git add filename

To add all files type git add .

Adding files to the staging area

5. Committing Changes

Next you have to commit your changes with a message describing the changes you've made to the file, use: git commit -m "Your commit message"

To view past commits:

git log

To revert to a particular commit:

git checkout commit_hash>

Committing changes

6. Creating a Branch

Branches

Branches are one of git features where it allow you to work on different versions of your project simultaneously. They are essential for managing features and fixes separately.

To create a new branch type:

git branch branch_name

Creating a branch

7. Viewing All Branches

To view all branches and see the active one type: git branch

Viewing all branches

8. Switching Active Branches

By default the active branch is Master to change that type: git switch branch_name

Switching branches

9. Merging a Branch

Merge changes from one branch into another example merging the created branch with the master branch with: git merge branch_name

Merging branches

10. Deleting a Branch

To delete a branch type: git branch -d branch_name

Deleting a branch

11.Have some Fun

Explore more advanced Git concepts like rebasing and stashing, or integrate with GitHub for remote version control and collaboration.
